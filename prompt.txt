Ich brauche einen Kalender mit dem ich zum einen einfache Termine verwalten kann und zum anderen komplexere Brauvorgänge abbilden kann.

Technische Voraussetzungen:
Die Applikation soll mit React gebaut werden und mit node 'v16.20.2' laufen.
Die Applikation soll ebenfalls mit apache2 funktionieren.
Die Applikation soll mit dem aktuellen Bootstrap gebaut werden.

Die Ansicht:
Per Default soll der Kalender angezeigt werden. Der Kalender soll die Termine und Brauvorgänge anzeigen.
Die Ansicht soll responsive sein und auf mobilen Geräten gut aussehen.
Die Ansicht soll dem Google Calendar ähnlichen.
In der Kalenderansicht soll zwischen folgenden Ansichten umgeschaltet werden können:
- Monatsansicht
- Wochenansicht
- Jahresansicht
- Dreimonatsansicht
- Tagesansicht

Ein Brauvorgang sieht etwa so aus:
Es wird ein Starttermin eingegeben und ausgewählt, ob obergärig oder untergärig. Dann schaut die Applikation selbst nach, ob ein Gärtank für die entsprechende Zeit zur Verfügung steht und ob am Ende der Zeit genug Fässer zur Verfügung stehen. Wenn ja, kann der Brauvorgang gespeichert werden. Wenn nein, wird eine entsprechende Meldung ausgegeben: "Kein Gärtank frei" oder "Nicht genug Fässer verfügbar".
Dazu ist es notwendig Gärtanks, Kühlschränke und Fässer mit zu verwalten.
Aktuell gibt es einen Kühlschrank, drei Gärtanks und 20 Fässer.
Pro Braugang wird ein Gärtank und 3 Fässer benötigt.
Gärtanks und Fässer können nicht doppelt belegt werden. Wenn also alle Tanks oder alle Fässe belegt sind und für einen weiteren Brauvorgang nicht zur Verfügung stehen, muss beim Neuanlegen oder Bearbeiten eines Brauvorgangs eine entsprechende Meldung ausgegeben und das Speichern verhindert werden.
Abhängig, ob obergärig (14 Tage) oder untergärig (10 Tage) gebraut wird, ist ein Gärtank 10 oder 14 Tage belegt und kann von keinem anderen Brauvorgang genutzt werden. Anschließend an diesen Zeitraum sind die drei Fässer für 10 oder 14 Tage belegt und können von keinem anderen Brauvorgang genutzt werden
Brauvorgänge sollen auch gelöscht werden können. Und zwar sowohl in der Bearbeiten Maske, als auch in der Kalenderansicht. Wird in der Kalenderansicht gelöscht, dann soll eine Sicherheitsabrage ausgegeben werden.

Die Daten:
Die Daten sollen im Json Format gespeichert werden.
Die Daten sollen in der Datei terminde.json gespeichert werden.
Es gibt einen eigenen Storage Service, der sich um die Daten kümmert. Dabei sollen die Datenquelle austauschbar sein. Per Default soll die Datei termine.json verwendet werden. Über eine Konfiguration soll aber auch eine Datenbank - MongoDB und MySQL - verwendet werden können. angesteuert werden können.


Helpers:
- Erstelle ein Script für Ubuntu im Verzeichnis /bin, mit dem eine zur Applikation passende MongoDB Datenbank erstellt wird.
- Erstelle ein Script für Ubuntu im Verzeichnis /bin, mit dem eine zur Applikation passende MySQL Datenbank erstellt wird.

Edit:
- Ein Klick auf einen Brauvorgang, egal in welcher Ansicht, soll eine Maske zum Bearbeiten öffnen.

Umdrücken:
- Innerhalb eines Brauvorgangs gibt es einen weiteren Termin "Umdrücken". Der Termin ist per Default entsprechend der Brauart (obergärig 14 Tage, untergärig 10 Tage) nach dem Brauvorgang. Der Termin soll bearbeitbar sein. Je nach dem ob er nach vorne oder hinten geschoben wird, verschiebt sich auch das Ende des Brauvorgangs.
- Wenn ich die Brauart ändere, dann soll sich automatisch der Termin für Umdrücken auch ändern.
- Das Umdrückdatum soll in der kalendarischen Ansicht innerhalb des Brauvorgangs hervorgehoben werden.
